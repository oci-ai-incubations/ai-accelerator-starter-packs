# .claude/ — Claude Code Configuration

This directory contains all Claude Code configuration for the AI Accelerator Starter Packs project: rules, skills (slash commands), tool permissions, and MCP server config.

## Rules (`rules/`)

Rules are automatically loaded into Claude's context when matching files are open or referenced.

| File | Scope | Purpose |
|---|---|---|
| `terraform.md` | `ai-accelerator-tf/**/*.tf`, schemas | Terraform conventions — directory structure, fmt, schema generation, versioning, category checklist |
| `testing.md` | `tests/**`, `schemas/tests/**` | Unit and schema test guidelines — mock providers, override_data requirements, expectation-driven schema tests |
| `oci.md` | `*.tf`, `iam_policies.md` | OCI conventions — default region, CLI profiles, ORM zips, IAM least-privilege policy reference |
| `playwright.md` | Always loaded | Playwright UI testing rules — interaction-first testing, video recording, error recovery, popup handling |

## Skills (`skills/`)

Skills are invoked as slash commands (e.g., `/deploy-and-test`). Each has a `SKILL.md` that defines its behavior.

| Skill | Command | Description |
|---|---|---|
| **setup** | `/setup` | Create an isolated sandbox (`/tmp/dat-sandbox-*/`) with all tools, OCI CLI config, and env vars needed by the deployment skills |
| **deploy-and-test** | `/deploy-and-test [category] [size]` | Full deployment workflow — GPU capacity check, region selection, ORM stack deploy, pod verification, and pack-specific API + UI testing via Playwright |
| **integration-test** | `/integration-test [category]` | ORM integration test lifecycle — schema gen, zip, stack create/update, plan, apply, pod verification, destroy |
| **update-stack** | `/update-stack` | Rebuild zip and update an existing ORM stack with latest code, then re-plan |
| **destroy-stack** | `/destroy-stack` | Destroy an ORM stack's infrastructure |
| **unit-test** | `/unit-test` | Run Terraform unit tests (plan-only, mocked providers, no cloud credentials) |
| **lint** | `/lint` | Run all Terraform linting — fmt, validate, tflint, checkov |
| **schema-gen** | `/schema-gen [category]` | Generate OCI Resource Manager schema for a starter pack category |
| **vss-test-coverage** | `/vss-test-coverage [section]` | VSS test specification — API endpoints, UI pages, user flows, and infra components with verification steps |

### Typical workflow

```
/setup                          # one-time: install tools, build env.sh
/deploy-and-test vss small      # deploy + test a starter pack
/destroy-stack                  # clean up when done
```

## Settings (`settings.json`)

Project-level tool permissions. Pre-approves commonly used commands so Claude doesn't prompt on every call:

- **Terraform/linting** — `terraform init/test/fmt/validate`, `tflint`, `checkov`, `pytest`
- **OCI CLI** — `oci resource-manager`, `oci ce`, `oci compute`, `oci db`, `oci os`, etc.
- **Kubernetes** — `kubectl get/describe/logs/config`
- **Git/GitHub** — `git status/diff/log/branch/add/checkout`, `gh pr/api`
- **Playwright MCP** — all `mcp__playwright__browser_*` tools (navigate, click, type, screenshot, etc.)
- **Shell utilities** — `ls`, `cat`, `jq`, `curl`, `cp`, `mkdir`, `base64`, etc.

## MCP Servers (`.mcp.json` in repo root)

Configures the Playwright MCP server for browser-based UI testing used by `/deploy-and-test`.

## Test Artifacts (`test-artifacts/`)

Gitignored. Stores Playwright video recordings (`.webm`) and test reports generated by `/deploy-and-test`.

## Setup: macOS notification hooks (optional)

Long-running skills like `/deploy-and-test` can take 30+ minutes. Add sound notifications to your **user-level** settings so you know when Claude finishes or needs permission.

Add the following to `~/.claude/settings.json`:

```json
{
  "hooks": {
    "Stop": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "osascript -e 'display notification \"Claude has finished\" with title \"Claude Code\" sound name \"Glass\"' & afplay /System/Library/Sounds/Glass.aiff"
          }
        ]
      }
    ],
    "PermissionRequest": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "osascript -e 'display notification \"Claude needs permission\" with title \"Claude Code\" sound name \"Ping\"' & afplay /System/Library/Sounds/Ping.aiff"
          }
        ]
      }
    ]
  }
}
```

- **Glass** sound plays when Claude finishes a task
- **Ping** sound plays when Claude needs permission approval
- Both also trigger a macOS notification banner

> **Note:** In VSCode extension mode, `PermissionRequest` hooks don't fire (permissions are handled by the extension UI). The `Stop` hook works in both terminal and VSCode.
